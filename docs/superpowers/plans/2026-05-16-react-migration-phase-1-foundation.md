# Phase 1: Foundation — React 19 Shell + Placeholder Routes

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Remove the SvelteKit + Svelte 5 stack entirely and replace it with a React 19 + Vite + TanStack Router + Zustand shell. All 22 tool routes render a `PlaceholderPage` component. App boots, types check, Tauri smoke passes.

**Architecture:** Vite is reconfigured with `@vitejs/plugin-react` and `@tanstack/router-vite-plugin` (file-based mode). The HTML entrypoint becomes `index.html` (Vite default) pointing at `src/main.tsx`, which mounts a React root with the TanStack Router provider. The file-based router scans `src/routes/__root.tsx` and `src/routes/<tool>.tsx`. SvelteKit-specific files (`+layout.svelte`, `+page.svelte`, `+layout.ts`, `app.html`, `svelte.config.js`) are deleted. `package.json` scripts are rewritten to drop `svelte-kit sync`, `svelte-check`, etc.

**Tech Stack (this phase):**

- React 19 + React DOM 19
- Vite 8 (kept)
- `@vitejs/plugin-react`
- `@tanstack/react-router` + `@tanstack/router-vite-plugin`
- Zustand (installed, not yet wired beyond placeholder)
- TypeScript 6.x (kept)

```typescript
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';
import { TanStackRouterVite } from '@tanstack/router-vite-plugin';
import path from 'node:path';

// Tauri expects a fixed port and host to bind the webview to.
const TAURI_DEV_HOST = process.env.TAURI_DEV_HOST;

export default defineConfig(async () => ({
	plugins: [
		TanStackRouterVite({
			routesDirectory: './src/routes',
			generatedRouteTree: './src/route-tree.gen.ts',
			autoCodeSplitting: true,
		}),
		react(),
	],
	resolve: {
		alias: {
			'@': path.resolve(__dirname, './src'),
			'@lib': path.resolve(__dirname, './src/lib'),
		},
	},
	clearScreen: false,
	server: {
		port: 1420,
		strictPort: true,
		host: TAURI_DEV_HOST || false,
		hmr: TAURI_DEV_HOST
			? {
					protocol: 'ws',
					host: TAURI_DEV_HOST,
					port: 1421,
				}
			: undefined,
		watch: {
			ignored: ['**/src-tauri/**'],
		},
	},
	envPrefix: ['VITE_', 'TAURI_ENV_*'],
	build: {
		target: process.env.TAURI_ENV_PLATFORM === 'windows' ? 'chrome105' : 'safari13',
		minify: !process.env.TAURI_ENV_DEBUG ? 'esbuild' : false,
		sourcemap: !!process.env.TAURI_ENV_DEBUG,
	},
}));
```

## Task 11: Rewrite validate-pages.ts for React routes

**Files:**

- Modify: `scripts/validate-pages.ts`

**Context:** The current script checks two things:

1. **Route file existence:** For each entry in `PAGES` (from `src/lib/services/pages.ts`), `src/routes/<id>/+page.svelte` must exist. In React it becomes `src/routes/<id>.tsx`.
2. **Tabs sync:** For each `PAGES` entry with `tabs`, the files `src/routes/<id>/tabs/<tab>-tab.svelte` must match.

Phase 1 deletes the tabs/ directories entirely (Task 8 removes them as part of the SvelteKit route purge). Tabs are reborn in Phase 5 with the new convention `src/routes/<id>/tabs/<tab>.tsx`. **For Phase 1**, the tabs branch is short-circuited — it will be re-enabled in Phase 5's plan.

- [ ] **Step 1: Replace the route-existence check from +page.svelte to .tsx**

Apply this minimal patch — edit `scripts/validate-pages.ts`:

```typescript
// before (line 134-137)
const routeExists = (pageId: string): boolean => {
	const routePath = join(ROUTES_DIR, pageId, '+page.svelte');
	return existsSync(routePath);
};

// after
const routeExists = (pageId: string): boolean => {
	const routePath = join(ROUTES_DIR, `${pageId}.tsx`);
	return existsSync(routePath);
};
```

And update the failure-message path at line 146:

```typescript
// before

// after
message: `Page "${page.title}" is registered in pages.ts but route file "src/routes/${page.id}.tsx" does not exist`,
```

```typescript

// after
```

- [ ] **Step 2: Short-circuit the tabs validation for Phase 1**

The `tabs/` directories are deleted in Phase 1 and reborn in Phase 5. To keep the validator green during Phases 1-4, replace `getActualTabIds` so it always returns an empty array — this turns `findMissingInPages`, `findMissingFiles`, `findTabsNotDefined`, and the `getPagesSummary` tab count all into no-ops.

Edit `scripts/validate-pages.ts`:

```typescript
// before (line 53-61)
const getActualTabIds = (pageId: string): readonly string[] => {
	const tabsDir = getTabsDir(pageId);
	return existsSync(tabsDir)
		? readdirSync(tabsDir)
				.filter((f) => f.endsWith('-tab.svelte'))
				.map(extractTabId)
				.filter((id): id is string => id !== null)
		: [];
};

// after
const getActualTabIds = (_pageId: string): readonly string[] => {
	// TODO(Phase 5): re-enable tab discovery once tabs are migrated to
```
